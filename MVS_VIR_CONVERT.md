# MVS 虚拟相机图片转换说明

## 虚拟相机目录结构

```
/var/tmp/VirtualCamera/Cameras/
├── Vir65253128/                  # 虚拟设备序列号目录
│   ├── MV-CS004-10UM.xml        # GenICam 设备描述文件
│   ├── MV-CS004-10UM_regfile.dat # 寄存器数值文件
│   ├── MV-CS004-10UM_strRegFile.dat
│   ├── Mono8/                   # 黑白（Mono）格式图片目录
│   │   └── *.bmp
│   └── RGB24/                   # 彩色格式图片目录
│       └── *.bmp
```

## 输入文件格式要求

| 项目 | 要求 |
|------|------|
| **图片格式** | 必须为 **BMP** |
| **分辨率** | 必须与虚拟设备寄存器配置的分辨率**完全一致** |
| **像素格式** | Mono8 目录 → 8-bit 灰度 BMP；RGB24 目录 → 24-bit RGB BMP |

## 分辨率来源

分辨率从设备目录下的 `*_regfile.dat` 文件中读取寄存器值获得：

| 设备 | 型号 | 分辨率 | Width 寄存器 | Height 寄存器 |
|------|------|--------|-------------|--------------|
| Vir65253128 | MV-CS004-10UM | 720×540 | 0x30360 = 720 | 0x303a0 = 540 |
| Vir2743340 | MV-CH120-10UM | 4096×3000 | 0x30360 = 4096 | 0x303a0 = 3000 |
| Vir11340209 | MV-CH120-10UC | 4096×3000 | 0x30360 = 4096 | 0x303a0 = 3000 |

**不推荐**从已有 BMP 文件推断分辨率，也不推荐使用相机型号的标称分辨率，以寄存器文件中的值为准。

## 寄存器文件格式

`*_regfile.dat` 为文本文件，每行一个寄存器地址-数值对：

```
<十六进制地址> <十进制数值>
30360 720       ← Width = 720
303a0 540       ← Height = 540
```

## 转换流程

1. 上传任意格式图片（jpg/png/bmp）
2. 选择目标虚拟设备 → 自动从 `_regfile.dat` 读取分辨率
3. 选择输出格式（Mono8 / RGB24 / Bayer8 / Raw）
4. 点击转换：
   - 图片自动缩放至设备分辨率（保持宽高比，黑边填充）
   - Mono8 → 转为 8-bit 灰度 BMP + 灰度调色板
   - RGB24 → 转为 24-bit RGB BMP（BGR 字节序）
   - Bayer8/Raw → 转为 8-bit 灰度 BMP
5. BMP 文件输出到 `{设备目录}/{格式目录}/{文件名}_时间戳.bmp`

## BMP 文件规范

### Mono8 (8-bit 灰度 BMP)

| 偏移 | 内容 |
|------|------|
| 0x00 | BITMAPFILEHEADER (14 bytes) = "BM" + 文件大小 + 数据偏移 |
| 0x0E | BITMAPINFOHEADER (40 bytes) = 宽/高/位深(8)/不压缩 |
| 0x36 | 调色板 (1024 bytes) = 256 个 BGRA 条目，灰度 R=G=B=0~255 |
| 0x436 | 像素数据 = 每像素 1 byte，bottom-up 行序，每行 4 字节对齐 |

### RGB24 (24-bit BMP)

| 偏移 | 内容 |
|------|------|
| 0x00 | BITMAPFILEHEADER (14 bytes) |
| 0x0E | BITMAPINFOHEADER (40 bytes) = 宽/高/位深(24)/不压缩 |
| 0x36 | 像素数据 = 每像素 3 bytes (B-G-R 顺序)，bottom-up 行序，每行 4 字节对齐 |
